# Biblioteca de bloques y plantilla base para el agente web de Actualícese ## Objetivo Este documento complementa el manual maestro del agente. Aquí se define una biblioteca base de bloques reutilizables y una plantilla inicial para que el agente construya páginas consistentes, rápidas de producir y compatibles con el ecosistema visual de Actualícese. El agente no debe empezar desde cero. Debe empezar desde una plantilla y ensamblar bloques. --- # 1. Filosofía de construcción ## Regla principal Cada página debe construirse con esta lógica: 1. elegir el tipo de página, 2. seleccionar una plantilla base, 3. insertar bloques reutilizables, 4. personalizar copy, imágenes, CTA y detalles, 5. agregar solo el CSS y JS estrictamente necesarios. ## Orden de trabajo obligatorio del agente Antes de escribir código, el agente debe decidir: * qué tipo de página es, * cuál es el CTA principal, * qué bloques necesita, * qué clases corporativas ya resuelven el diseño, * qué CSS adicional debe encapsular. --- # 2. Plantilla base universal Esta es la estructura madre que debe usar el agente como punto de partida. ## Estructura general recomendada ```html

Bloques reutilizables obligatorios ## Bloque 1. Hero comercial 50/50 **Uso:** productos, promociones, suscripciones, herramientas, eventos. ### Estructura ```html

Texto de apoyo

Título principal con énfasis

Descripción breve del beneficio principal.

Quiero más información
``` ### Cuándo usarlo * cuando hay una oferta clara, * cuando el usuario debe entender el beneficio en los primeros segundos, * cuando hay apoyo visual fuerte. --- ## Bloque 2. Encabezado de sección **Uso:** iniciar cualquier bloque importante. ```html

Texto breve que introduce la sección.

``` ### Regla No debe usarse un bloque largo sin un encabezado visible y fácil de escanear. --- ## Bloque 3. Beneficios en tarjetas **Uso:** beneficios, módulos, entregables, razones de compra. ```html

Beneficio 1

Descripción corta y útil.

Descripción corta y útil.

Beneficio 3

Descripción corta y útil.

``` ### Regla Máximo 3 o 4 tarjetas por fila visual. Nada de zoológico de cajas infinitas. --- ## Bloque 4. Imagen + lista de valor **Uso:** explicar por qué comprar, qué incluye o cómo funciona. ```html

Lo que incluye tu compra

  • Punto clave 1
  • Punto clave 2
  • Punto clave 3
  • Punto clave 4
``` --- ## Bloque 5. Comparativo simple **Uso:** planes, versiones, antes/después, incluido/no incluido. ```html

Característica Opción A Opción B

Elemento 2 Incluido Adicional
``` ### Regla Las tablas deben ser pocas, legibles y con versión responsive simple. --- ## Bloque 6. FAQ / acordeón **Uso:** objeciones, preguntas frecuentes, soporte comercial. ```html

Preguntas frecuentes

Respuesta clara y concreta.

Respuesta clara y concreta.

``` --- ## Bloque 7. CTA final **Uso:** cierre fuerte de la página. ```html

Da el siguiente paso hoy

Texto corto que refuerza la acción.

Comprar ahora
``` ### Regla Toda landing o página promocional debe cerrar con un CTA claro. No dejar al usuario flotando en el vacío cósmico del scroll. --- # 4. Bloques opcionales recomendados ## Bloque 8. Cifras o indicadores Útil para confianza, resultados o contexto. ## Bloque 9. Testimonios Solo si hay contenido real. Nada de muñecos de cartón con frases genéricas. ## Bloque 10. Agenda o cronograma Muy útil para eventos y capacitaciones. ## Bloque 11. Perfil del conferencista Para páginas de evento. ## Bloque 12. Navegación interna con anclas Para especiales editoriales o páginas largas. --- # 5. CSS base sugerido para complementar el sistema Este CSS no reemplaza los estilos corporativos. Solo ayuda a los bloques nuevos. ```css .actlp-page { width: 100%; } .actlp-section { width: 100%; } .actlp-img-fluid { display: block; max-width: 100%; height: auto; } .actlp-cards-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px; margin-top: 32px; } .actlp-card { padding: 24px; } .actlp-table-wrap { overflow-x: auto; margin-top: 24px; } .actlp-table { width: 100%; border-collapse: collapse; } .actlp-table th, .actlp-table td { padding: 14px 16px; text-align: left; } .actlp-faq-item { border-bottom: 1px solid #d9d9d9; } .actlp-faq-question { width: 100%; padding: 18px 0; background: none; border: none; text-align: left; font: inherit; cursor: pointer; } .actlp-faq-answer { display: none; padding-bottom: 16px; } .actlp-faq-item.is-open .actlp-faq-answer { display: block; } @media (max-width: 991px) { .actlp-cards-grid { grid-template-columns: 1fr 1fr; } } @media (max-width: 767px) { .actlp-cards-grid { grid-template-columns: 1fr; } } ``` --- # 6. JS base sugerido ```javascript document.addEventListener('DOMContentLoaded', function () { var faqButtons = document.querySelectorAll('.actlp-faq-question'); faqButtons.forEach(function (button) { button.addEventListener('click', function () { var item = button.closest('.actlp-faq-item'); item.classList.toggle('is-open'); }); }); }); ``` ### Regla El agente solo debe incluir JS base cuando el bloque lo necesita. Si no hay interacción, debe entregar un `script.js` vacío o con comentario mínimo. --- # 7. Plantillas maestras por tipo de página ## Plantilla A. Landing de producto Bloques recomendados: 1. Hero comercial 50/50 2. Beneficios en tarjetas 3. Imagen + lista de valor 4. Comparativo o detalle 5. FAQ o prueba social 6. CTA final ## Plantilla B. Ruta editorial Bloques recomendados: 1. Hero informativo 2. Navegación interna 3. Secciones de contenido con encabezado 4. Recursos destacados 5. CTA contextual ## Plantilla C. Evento o capacitación Bloques recomendados: 1. Hero del evento 2. Datos clave 3. Qué aprenderás 4. Perfil del conferencista 5. Agenda o metodología 6. CTA final ## Plantilla D. Promo simple Bloques recomendados: 1. Hero directo 2. Oferta o beneficio 3. Razones para actuar 4. CTA final --- # 8. Instrucción adicional para el agente Pega también esto como parte de sus reglas de trabajo: ```text No empieces una página desde cero. Primero elige una plantilla maestra. Luego selecciona bloques reutilizables y arma la página con ellos. Cuando construyas una página, usa primero estos bloques: hero comercial 50/50, encabezado de sección, tarjetas de beneficios, imagen + lista, comparativo simple, FAQ y CTA final. No inventes estructuras nuevas si una plantilla existente ya resuelve el objetivo. ``` --- # 9. Qué sigue después de este documento Después de esta biblioteca, el siguiente nivel para dejar el agente realmente fino es crear: 1. un archivo de prompts de uso interno para el equipo, 2. una biblioteca de copys estructurados por tipo de bloque, 3. una guía de naming para clases nuevas, 4. una checklist de QA antes de publicar, 5. una plantilla de brief para pedir páginas al agente. Eso ya convierte el agente en una línea de ensamblaje bien aceitada, no en una ruleta con HTML.
